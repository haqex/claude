# Proposals

Pending repo changes generated by pipelines — new skills, new pipelines, new reference docs. Nothing here is applied until you explicitly approve it.

## Filename Format

```
YYYY-MM-DD-<skill-or-artifact-name>.md
```

Example: `2026-03-09-claude-code-plugin-setup.md`

## Status Values

| Status | Meaning |
|--------|---------|
| `pending` | Waiting for review |
| `approved` | Applied to the repo |
| `rejected` | Discarded |

## Approving a Proposal

```
/project:approve-proposal 2026-03-09-claude-code-plugin-setup
```

This reads the proposal, writes the proposed files to their target locations, and marks the proposal as `approved`.

## Rejecting a Proposal

Just delete the file, or edit the frontmatter `status` to `rejected`. No files are written.
